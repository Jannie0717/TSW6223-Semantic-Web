3.0 Solution Development
========================

3.1 System Overview
-------------------
Solution 1 is a Semantic Web-Based Course Recommendation System for the education domain. The system helps students identify suitable courses based only on their selected interest area. The Current Skills selection and Career Path / Skill Gap Analysis features are not included in this version because Solution 1 focuses only on course recommendation.

The system follows this flow:
Student Interest -> RDF/RDFS Knowledge Base -> SPARQL Query -> Recommended Courses

3.2 Selected Semantic Web Technology
------------------------------------
The selected topic for Solution 1 is Category 2: RDF, RDFS and SPARQL.

RDF is used to represent education data as triples. For example, a course can be linked to the skill it teaches and the interest area it belongs to. RDFS is used to define the structure of the data, including classes such as Student, Course, Skill and Interest. It also defines properties such as interestedIn, teachesSkill, relatedToInterest and hasPrerequisiteSkill. SPARQL is used to query the RDF graph and retrieve recommended courses based on the selected interest area.

3.3 RDF/RDFS Data Design
------------------------
The RDF/RDFS data file is stored in:

data/education_rdf_rdfs.ttl

Main RDFS classes:
- Student
- Course
- Skill
- Interest

Main RDF properties:
- interestedIn: links Student to Interest
- teachesSkill: links Course to Skill
- relatedToInterest: links Course to Interest
- hasPrerequisiteSkill: links Course to prerequisite Skill
- difficulty: stores course difficulty
- creditHour: stores course credit hour

3.4 Application Implementation
------------------------------
The application is implemented using Python Flask. RDFLib is used to load the Turtle file and execute SPARQL queries. The website interface is created using HTML and CSS.

Main files:
- app.py: Main Flask application and SPARQL logic
- data/education_rdf_rdfs.ttl: RDF/RDFS knowledge base
- templates/index.html: User interface
- static/style.css: Website styling
- run_tests.py: Functionality testing script

3.5 Course Recommendation Process
---------------------------------
The user only selects an interest area. The system creates a temporary RDF triple for the current student and links the student to the selected interest. Then, a SPARQL query searches for courses that are related to the selected interest area.

Example logic:
1. Student selects Artificial Intelligence.
2. The system stores the selected interest as an RDF triple.
3. The system searches for AI-related courses using SPARQL.
4. The system recommends courses such as Introduction to Python Programming, Machine Learning, Deep Learning, Algorithm Design and Statistics for Data Analytics.

3.6 Key Features
----------------
- Interest area selection only
- RDF/RDFS model preview
- SPARQL-based course recommendation
- Course difficulty and credit hour display
- Skills taught by each course
- Prerequisite skill display
- SPARQL query example for demonstration
