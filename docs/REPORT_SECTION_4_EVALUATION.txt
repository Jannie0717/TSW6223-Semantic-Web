4.0 Evaluation
==============

4.1 Evaluation Approach
-----------------------
Functionality testing was conducted to ensure that the Solution 1 application meets its requirements. The testing checks whether the RDF/RDFS data can be loaded, whether interest areas can be retrieved, whether the RDFS model preview works, and whether SPARQL queries return suitable courses based on the selected interest area.

4.2 Testing Tool
----------------
The automated testing script is stored in:

run_tests.py

The script can be executed using:

python run_tests.py

4.3 Tested Functions
--------------------
The following functions were tested:
- RDF/RDFS graph loading
- Retrieval of interest areas
- RDFS class and property preview
- SPARQL course recommendation query
- Interest-only course recommendation output

4.4 Test Result Summary
-----------------------
All test cases passed. The system successfully loads the RDF/RDFS data file, retrieves the required education data, and recommends suitable courses using SPARQL.

Expected output after running the test script:

PASS - RDF/RDFS schema and data loaded
PASS - AI course recommendation by interest only
PASS - Data Science course recommendation by interest only
PASS - Cybersecurity course recommendation by interest only
PASS - Web development course recommendation by interest only

All Solution 1 interest-only course recommendation tests passed.

4.5 Evaluation Conclusion
-------------------------
The evaluation shows that Solution 1 works as intended. The system applies Category 2 Semantic Web technologies correctly by using RDF to store education data, RDFS to define data structure and SPARQL to retrieve suitable course recommendations based on the selected interest area.
